# AI code corpus — MANIFEST

Added in 0.10.0 for code-mode calibration. Each sample is unedited assistant-style
output for the kind of request that produces it (Claude Fable 5, writing as the named
assistant archetype would with no style instructions). Two honesty notes, because a
corpus generated by the thing being detected invites circularity:

1. The samples were written as a typical chat answer to the simulated prompt, not
   assembled tell-by-tell from the rule list. The tells they carry are the ones the
   external research catalog (see `eval/README.md`) documents for each archetype, so
   the anchor is the literature, not the engine.
2. `13-agentic-terse-cache.py` is a deliberate hard case: agentic CLI assistants are
   prompted toward few comments and terse style, and the research is blunt that
   surface analysis can't reliably catch that register. It's expected to score 0 and
   the floors are set with that miss included, so nobody can tune the engine to a
   corpus that pretends the hard case doesn't exist.
